## 文档纪律

- 架构文档是唯一的设计事实源；代码注释里引用章节号（如 §5.2）。
- 重大设计变更先写 ADR（编号递增，永不删除，被推翻的标记 Superseded）。
- 调研报告按 `YYYY-MM-主题.md` 命名，保留当时的数据与链接，不回头改写。
