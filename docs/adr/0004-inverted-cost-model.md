# ADR-0004 成本模型反转：为"token 免费、延迟贵"设计

| | |
| --- | --- |
| 状态 | Accepted |
| 日期 | 2026-07-02 |

## 背景

所有主流 agent（Claude Code / codex / goose / OpenClaw）诞生于云端计费模式，
隐含假设"token 昂贵，按需使用、用完即停"。本地推理的真实成本结构相反：
闲置 GPU 的边际 token 成本以电费毛计，而 prefill 延迟（本地体验头号杀手）
无法用钱买回。没有 agent 为反转后的目标函数设计过。

## 决策

把"token 免费、延迟是黄金"作为 Kestrel 的第一性成本假设，允许任何
以 token 换延迟/换质量的设计，只要它不违反前缀稳定性（原则 1）与权限门（原则 5）。

直接产物（进路线图）：

1. **投机代理**（M4）：空闲 slot 预计算用户最可能的下一步（继续/批准后续/常见指令），
   命中秒回、失手丢弃。是影子槽预热（架构文档 §7）从"压缩"到"任意未来"的推广。
2. **夜班**（M5）：闲时自主家务——记忆蒸馏、索引重建、探针复跑、草稿起草。
   安全边界：默认只读、权限门管辖、零网络监听（OpenClaw 教训的三重否定）。

北极星级延伸（睡眠周期 / 本地 LoRA 自我进化）见 [vision.md](../vision.md)，
落地前另立 ADR。

## 被否决的替代

- **维持云端式节俭**：放弃本地最大的结构性优势，产品沦为"离线版 XX"。
- **无限自治（AutoGPT 式）**：烧 token 不等于放开自治；投机与夜班的所有动作
  仍受迭代上限、预算闸、权限门约束，且投机结果不落盘、夜班默认只读。

## 影响

- 模型池调度器需要"闲时作业"优先级层（前台 turn 永远优先，投机/夜班可被瞬时抢占）。
- 机组账本增加投机命中率与夜班产出的统计（把免费 token 的价值可视化）。
- 功耗/温度是新的资源约束：夜班需要可配置的功耗上限与时段窗口。

## 重开条件

- 用户实测表明投机命中率过低（< 30%）且预热干扰前台延迟 → 收缩投机范围。
- 家用电价/散热约束普遍高于预期 → 夜班默认改为手动触发。
