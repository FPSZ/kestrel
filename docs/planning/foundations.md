# Kestrel · 地基约定清单

> 这份清单收录**横切、难回填**的工程约定——"现在几行、以后重构一周"那一类。
> 灵感来自 i18n：若等代码铺开再改，要动几百个调用点。所以趁早锁进规范。
>
> 本文件是**跟踪表**，非事实源：每条一旦定案，硬规则上升到 [AGENTS.md §5](../../AGENTS.md)，
> 带取舍的架构决策落一个 [ADR](../adr/)，本表对应行标注归属。分三档按"回填痛苦 × 波及面 ×
> 锁定成本"排序。

**状态**：`已定`（用户拍板或被铁律逼死）· `建议默认`（我给了默认，待你确认）· `需 ADR`（要单独决策记录）。

## Tier 1 —— 必须现在锁（波及全局、回填惨烈、锁定几乎白送）

这一档全部上升为 [AGENTS.md §5 地基铁律](../../AGENTS.md)；违反即拒绝，与"禁 emoji""禁破前缀"同级。

| # | 约定 | 不定的后果 | 决策 / 默认 | 状态 | 归属 |
| --- | --- | --- | --- | --- | --- |
| 1 | **i18n 用户可见文本**（模型侧英文豁免） | 硬编码字符串铺满两个前端 + core，回填要审每个渲染点 | 表现层本地化；zh-CN+en-US；英文 key；en-US 回退 | 已定 | ADR-0008 |
| 2 | **语言中立事件日志**（只存 code+params，不存句子） | 回放 fixture 变语言相关、换语言要改历史 | 事件携带稳定 code + 结构化参数 | 已定 | ADR-0008 |
| 3 | **结构化错误分类**（`ErrorCode`+params，不产句子） | core 里散落几百个英文错误串，既不可测也不可本地化 | 定义稳定错误枚举，边缘渲染 | 已定 | ADR-0008 |
| 4 | **设计令牌**（无硬编码颜色/间距/字号/圆角） | 换主题/做浅色模式要改每个组件——视觉版的硬编码地狱 | 所有视觉值进 `@theme` 令牌；组件只引令牌 | 已定 | AGENTS §5（视觉版 i18n） |
| 5 | **core 确定性** | 事件流里混入时钟/随机，回放 harness 直接失效——测试地基塌 | 禁 wallclock/random/env 读进影响事件流的 core 路径；非确定性作为**数据**在 IO 边缘捕获后进入 | 已定 | AGENTS §5 |
| 6 | **事件日志前向兼容** | 第一次改 `Event` 形状就让所有人历史会话 + 全部 fixture 反序列化失败 | `schema_version` 信封 + 容忍解析（未知字段/变体不 fail）+ **只增不改不复用**字段 | 已定（ADR-0011） | AGENTS §5 + ADR-0011 |
| 7 | **密钥处理** | api_key 之类混进事件日志/tracing/UI/提交，回填要审每个序列化与打印点 | `SecretString` 型（不 Debug 打印）；只从 env / OS store 取；永不入事件日志/日志/UI/提交；显示脱敏 | 已定（强化安全红线） | AGENTS §5 + §7 |
| 8 | **时间戳** | 本地时间入库、按语言手拼日期，回放断言随时区飘 | UTC 存、边缘按 locale 格式化、回放断言排除时间戳 | 已定 | AGENTS §5（配合 ADR-0008） |
| 9 | **日志 vs UI 分离** | "日志英文"和"UI 本地化"混为一谈，要么日志被翻译、要么 UI 硬编码英文 | 开发日志（tracing）英文/结构化/脱敏；用户可见文本走 catalog | 已定 | AGENTS §5（细化） |

## Tier 2 —— 尽快定（中等痛苦，部分有真取舍）

| # | 约定 | 不定的后果 | 建议默认 | 状态 |
| --- | --- | --- | --- | --- |
| 10 | **配置前向兼容 + 优先级** | 重蹈 OpenHands 140 字段配置怪物；加字段就破旧配置 | 优先级 `默认 < 文件 < env < CLI flag < 会话`；未知键 warn 不 fail；单一 typed 结构、保持小而平；env 变量统一前缀 `KESTREL_` | 建议默认 |
| 11 | **WebUI 可访问性基线** | 建完 30 个组件再补 a11y 是重活；磨砂"呼吸"动画晃到人 | 语义 HTML + 键盘可达 + 可见焦点环 + 尊重 `prefers-reduced-motion`（关磨砂动画）+ 达标对比度 | 建议默认 |
| 12 | **存储位置 / 布局 + 迁移** | 硬编码 `./sessions`，以后搬到 OS app-data 要给所有人做迁移 | OS 标准目录（`directories` crate）+ 版本化迁移；`.kestrel/` opt-in 保留"手边数据" | 已定（ADR-0009） |
| 13 | **跨平台路径 + 工作目录沙箱** | 路径逃逸漏洞 + Windows/POSIX 语义不一致，散落各工具 | 统一规范化/canonicalize；拒绝越出 workdir（安全红线）；处理 Windows 长路径/大小写；集中一处校验 | 建议默认 |
| 14 | **server↔console 线协议版本化** | 契约一变就悄悄打断前端；无兼容策略 | `/api/v1` 命名空间；wire 用 snake_case（Rust 原生）TS 侧镜像；破坏性变更升版本 | 建议默认 |
| 15 | **稳定 ID 策略** | 会话/工具调用 ID 用随机 UUID → 破坏回放确定性；不可排序 | 事件 `seq` 已单调；跨会话/工具调用关联 ID 用**确定性、可排序**方案（每会话单调或从事件派生），非 `Math.random` | 建议默认 |
